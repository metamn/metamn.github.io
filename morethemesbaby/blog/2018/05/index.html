Lately I’ve been working on the pro version of Log Lolla to make it a theme which enables knowledge creation. Since a while I’m stuck with it, procrastinated, and adding more issues / gotchas / requirements than code.

Creating knowledge is a hard problem. I have 6 months of data on this blog, around 200 posts, but still unable to figure out what knowledge lies in here.

So the strategy is the following:

  1. Continue adding posts, and see how knowledge can be distilled.
  2. Create a new Pro version of the Log Lolla theme focusing on presentation instead of knowledge.

More Themes Baby is in the loop

Post featured image

Post content for Standard Post format

The Front-End Tooling Survey 2018 Results is out and we are glad to see all technologies we use to create themes are in line with the majority.

  • SASS / SCSS — 65.33%
  • No CSS framework (33.16%) and Custom CSS framework (15.17%) vs. Bootstrap (34.69%)
  • BEM as CSS methodology and naming scheme — 38.85%
  • Autoprefixer  for cross browser compatibility — 57.45%
  • Flexbox and Grid as emergig CSS features — 68.04% and 17.95%
  • Gulp as task runner — 29.70%, second place after NPM
