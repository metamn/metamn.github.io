How Hosting Works?

Web hosting means, in layman terms, various technological layers between the visitor and the content. Less layers the faster the experience. What HardyPress does is eliminating all layers and exposing content directly to visitors.

This is the so called static site hosting — when a dynamic site like WordPress is compiled into static files.

A visitor browsing a static site is served immediately with the content ; when browsing a dynamic site the content is assembled on-the-fly with business logic applied to database queries. That takes time, lots of time.

Static sites

They have been around since 2008 when Github’s Tom Preston Werner created Jekyll, the first static site generator.

Since then everybody tries to serve its content using a static site.

Modern web technologies like Single Page Applications (SPA) and Progressive Web Applications (PWA) when used as Content Management Systems (CMS) they all do server side rendering, which means, they deploy a static site instead of a dynamic site.

Content Delivery Networks (CDN) and premium hosting services like WP Engine are trying to do the same using a more complicated and already obsolete technology called caching.

Headless CMS

From static sites we’ve got immediately and easily to the Headless CMS scene, a hot trend lately, which means the Content Management System deals only with the backend and forgets the frontend.

In other words a Headless CMS deals only with creating and managing content and does not care how the content is presented to the visitor. In most cases as a static site, generated by a static site generator.

What HardyPress Does?

In short, transforms WordPress into a Headless CMS and generates a static site.

You still have all the advantages of a usual WordPress — the standard admin interface — but the speed of your site becomes the fastest possible on the market today.
